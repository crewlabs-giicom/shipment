💡 Cara Kerja Integrasi Sheets
📋 Header yang dibaca dari Rekap Kontainer:
NO KONTAINER  ·  TANGGAL PESAN  ·  TANGGAL STUFFING  ·  ESTIMASI TANGGAL BERANGKAT  ·  ESTIMASI SAMPAI PORT  ·  TAGS  ·  SKU  ·  QTY  ·  AMOUNT  ·  PPB  ·  SUPPLIER  ·  VENDOR  ·  TANGGAL DATANG  ·  PBM
📊 Penentuan Status Otomatis:
• Kolom TANGGAL DATANG atau PBM terisi → Delivered
• Ada SKU + QTY + TANGGAL PESAN (dengan/tanpa kontainer) → Production
• Ada TANGGAL STUFFING → Stuffing atau lebih tinggi
• Status Delivered / Cleared yang sudah diset manual tidak akan ditimpa
🔄 Cara Kerja Merge:
• PO + Kontainer + Item sama → 1 baris (qty & value dijumlah)
• PO tanpa kontainer → masuk sebagai Production (belum stuffing)
• Saat kontainer sudah ada di Rekap → otomatis ter-update
• Catatan & dokumen yang diisi manual tidak akan ditimpa
🚛 Forwarder & Supplier:
• Dibaca dari kolom SUPPLIER dan VENDOR
• Forwarder kosong → diisi otomatis dari Aturan Forwarder di Master Data
• Berbasis kode No. Kontainer (contoh: AIR → AIR, AJKY → HARVEST-AIR)
⚙️ Update Apps Script:
• Setelah update file apps-script-sheets-sync.gs
• Buka Apps Script → Deploy → Manage deployments
• Edit → pilih versi baru → Deploy
• URL tidak berubah — tidak perlu update di aplikasi
